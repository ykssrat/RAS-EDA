# 电路可视化方案与分割规则汇总

## 第一部分：电路图生成方案对比分析

针对论文发表及不同规模电路（10~1000节点）的可视化需求，以下是四种主流方案的对比分析：

| 方案 | 格式 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **1. Mermaid (当前方案)** | `.md` | - **轻量级**，直接嵌入 Markdown。<br>- 文本格式，易于生成和版本控制。<br>- VS Code 预览方便。 | - **布局算法较弱**，节点多时（>50）线条混乱。<br>- 缺乏专业的电路符号（只是方框和圆圈）。<br>- 难以导出高质量矢量图用于论文。 | 小规模电路 (<50节点) 的快速预览和调试。 |
| **2. Graphviz (DOT)** | `.dot` / `.svg` | - **布局算法强大**，适合大规模有向图。<br>- 自动化程度极高。<br>- 导出 PDF/SVG 矢量图质量高，适合论文。 | - 默认样式比较“计算机科学”，不像标准电路图。<br>- 需要安装 Graphviz 工具链。 | 中大规模电路 (50-500节点) 的拓扑结构分析。 |
| **3. Netlist SVG / Yosys** | `.svg` | - **专业的电路原理图风格**（逻辑门形状）。<br>- 工业界常用工具链。<br>- 视觉效果最接近教科书。 | - 配置复杂，通常需要依赖 Yosys 等 EDA 工具。<br>- 纯 Python 实现较难，需要调用外部命令。 | 追求极致视觉效果的论文插图。 |
| **4. Gephi / Cytoscape** | `.gexf` / `.graphml` | - **交互式分析**，适合超大规模网络。<br>- 强大的聚类和社区发现可视化。<br>- 适合展示“分割效果”而非“电路逻辑”。 | - 需要专门的软件打开，无法直接在文档中预览。<br>- 主要是网络分析视角，而非电路视角。 | 大规模电路 (>500节点) 的分割算法效果展示。 |

### 论文建议方案 (分层策略)

1.  **小规模示例 (s27, s382等)**：
    *   **推荐：Graphviz (DOT) + 自定义形状**。
    *   **理由**：Mermaid 对于 s382 这种近百个节点的电路已经显得力不从心。Graphviz 可以生成紧凑的布局，并且可以通过设置 `shape` 和 `style` 来模拟电路门（如 AND 用梯形，DFF 用矩形）。生成的 SVG 可以直接插入 LaTeX。

2.  **大规模电路 (>500节点)**：
    *   **推荐：Gephi (导出图片)** 或 **Matplotlib (NetworkX)**。
    *   **理由**：对于大电路，画出每一根连线已经没有意义。论文中通常展示的是**宏观结构**（如模块间的连接密度）或**分割后的簇（Cluster）**。建议使用 Python 的 `networkx` 库绘制节点分布图，用不同颜色标记分区 A 和 B，直观展示分割边界。

## 第二部分：FM算法电路分割规则

### 核心准则
1. **禁止自动运行**：AI不得主动执行代码，仅提供操作指引。
2. **禁止调用终端**：严禁使用 `run_in_terminal` 等执行工具。
3. **文件命名**：修改代码时若需备份，使用 `_1`, `_2` 等后缀。

### 生成规则 (基于信号流向)
1. **原电路输入 (PI)**：初始默认分配给 A 区；若 B 区实例也使用该输入，则 B 区也声明为 `input`。
2. **原电路输出 (PO)**：归属于**驱动该信号的实例**所在的分区。
3. **割线定义**：连接 A 区与 B 区的内部信号视为割线。
4. **割线方向 (动态决定)**：
   - **A $\to$ B**：若信号源实例在 A，则 A `output`, B `input`。
   - **B $\to$ A**：若信号源实例在 B，则 B `output`, A `input`。
5. **命名规范**：割线端口必须添加 `cut_` 前缀（如 `cut_G17`），内部信号保持原名并通过 `assign` 连接。
6. **端口完备性**：生成的 Verilog 模块端口列表需包含：本区用到的 PI、本区产生的 PO、以及所有进出本区的 `cut_` 信号。
7. **格式规范**：graphghml生成的Verilog 文件中所有 `input` 声明在一行，所有 `output` 声明在一行。
8. **语法规范**：生成的 Verilog 代码必须符合 IEEE Verilog 语法规范（例如模块实例化端口列表必须包含在括号内）。
   
### s27电路特例
- **输入**：`CK, G0, G1, G2, G3` -> A区 `input`。
- **输出**：`G17` -> B区 `output`。
- **端口完备性**：分割后的端口集合必须覆盖原电路所有端口及割线端口。

### 项目结构与运行
#### 核心模块
1. `py/FM_part.py`: FM算法核心，生成 `.graphml` 分割文件。
2. `py/G_to_v.py`: 读取 `.graphml`，生成 Verilog (`_a.v`, `_b.v`)。
3. `py/window.py`: 交互式主程序。

#### 运行方式
**推荐**：一、运行 `py/window.py`
```bash
python py/window.py
```
1. **电路分割**：输入 `s27` -> 生成 `output/s27.graphml`
2. **生成Verilog**：输入 `s27` -> 生成 `output/s27_a.v` 和 `output/s27_b.v`

二、**独立运行**：
- 分割：`python py/FM_part.py s27`
- 生成：`python py/G_to_v.py s27`

### 一些说明
1. **语法**：使用 `assign G11 = cut_G11;` 这样的语法，`cut` 前缀其实是为了说明该信号被切割了，但是不加前缀是为了方便例化并且说明了信号的流向。
