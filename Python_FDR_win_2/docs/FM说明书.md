# 大规模Verilog电路分割系统

## 概述

本文档介绍了使用FM(Fiduccia-Mattheyses)算法对大规模Verilog电路进行分割的新功能。该功能旨在将大型电路划分为较小的部分，以便提高仿真效率、降低内存消耗，并可能支持并行处理。

## 功能说明

### 1. Verilog解析器 (VerilogParser)

VerilogParser类负责解析Verilog源文件并提取电路结构信息：

- **模块识别**: 从Verilog文件中提取所有模块定义
- **实例提取**: 识别模块中的子模块实例
- **连接分析**: 分析模块内部的信号连接关系
- **网络提取**: 提取wire和端口信息

### 2. 图表示 (Graph)

Graph类将电路表示为图结构，便于应用图分割算法：

- **节点表示**: 每个电路实例表示为图中的节点
- **边表示**: 实例间的连接关系表示为图中的边
- **权重管理**: 支持节点和边的权重设置
- **动态更新**: 支持动态添加和删除节点及边

### 3. FM分割算法 (FMPartitioner)

FMPartitioner类实现了经典的FM图分割算法：

#### 算法原理
FM算法是一种基于贪心策略的图分割算法，通过迭代优化来最小化分割代价：

1. **初始化**: 将图节点随机分配到两个分区
2. **增益计算**: 计算每个节点移动到另一分区的增益
3. **节点选择**: 选择具有最大增益的节点进行移动
4. **迭代优化**: 重复上述过程直到收敛

#### 特性
- **负载均衡**: 通过balance_factor参数控制两分区大小平衡
- **桶排序**: 使用桶数据结构加速增益最大的节点查找
- **锁定机制**: 防止节点在同一次迭代中被多次移动
- **迭代优化**: 多次迭代寻找最优解

### 4. 电路分割器 (CircuitPartitioner)

CircuitPartitioner类整合了整个分割流程：

- **端到端处理**: 从Verilog解析到分割结果输出的完整流程
- **参数配置**: 支持自定义分割参数
- **结果导出**: 将分割结果保存为JSON格式文件

## 使用方法

### 基本用法

```python
# 创建电路分割器实例
partitioner = CircuitPartitioner('path/to/verilog/file.v')

# 执行电路分割
partitions = partitioner.partition_circuit(balance_factor=0.5)

# 保存结果到文件
partitioner.save_partitions(partitions, 'partition_result.json')
```

### 参数说明

- `verilog_file`: 待分割的Verilog文件路径
- `balance_factor`: 分区平衡因子，控制两分区大小差异 (默认0.5)
- `max_iterations`: FM算法最大迭代次数 (默认10)

### 输出格式

分割结果以JSON格式保存，包含以下信息：

```json
{
  "partition_a_count": 15,
  "partition_b_count": 12,
  "cutsize": 8,
  "partition_a_nodes": ["module.inst1", "module.inst2", "..."],
  "partition_b_nodes": ["module.inst3", "module.inst4", "..."]
}
```

## 技术优势

### 1. 内存效率
- 将大规模电路分解为较小部分，降低单次处理的内存需求
- 支持流式处理大型电路文件

### 2. 计算优化
- 使用FM算法确保高质量的分割结果
- 最小化跨分区连接，减少通信开销

### 3. 灵活配置
- 可调节的平衡因子满足不同应用场景需求
- 模块化设计便于扩展和定制

## 应用场景

### 1. 大型电路仿真
将大规模电路分割为多个小块，分别仿真后再合并结果，提高仿真效率。

### 2. 并行处理
利用分割结果实现电路的并行处理，充分利用多核处理器性能。

### 3. 故障诊断
通过分割隔离问题区域，提高故障定位和调试效率。

### 4. 物理设计
在布局布线阶段指导模块分布，优化芯片物理设计。
