# 电路切换说明书

本文说明如何在工程中切换仿真电路（以 s27 ↔ s382 为例），并给出 end_time 的确定方法。

## 1. 更新文件清单 verilog_file.f
编辑 `circuit/verilog_file.f`，仅保留目标电路的文件行。

- 切换到 s27 示例：
  ```
  ./circuit/s27.v
  ./circuit/s27_tb.v
  ./circuit/stdcells.v
  ```
- 切换到 s382 示例：
  ```
  ./circuit/s382.v
  ./circuit/s382_tb.v
  ./circuit/stdcells.v
  ```

## 2. 更新 config.json
在工程根目录修改 `config.json`，至少同步以下字段：
- `circuit_name`: 目标电路名（如 `"s382"`）。
- `clk_name`: testbench 中的时钟信号名（s27/s382 均为 `"CK"`）。
- `clk_period`: 时钟周期，单位与 testbench 延时一致（s27/s382 示例均为 `10`，对应 `#5` 翻转）。
- `end_time`: 仿真截止时间，见下节计算方式。
其余路径字段一般保持默认（`./output/*.json`, `./run.tcl`）。

示例（s382）：
```json
{
  "circuit_name": "s382",
  "clk_name": "CK",
  "clk_period": 10,
  "end_time": 175,
  "circuit_info_file": "./output/circuit_info.json",
  "golden_file": "./output/golden.json",
  "fault_file": "./output/fault.json",
  "path": "./",
  "tcl_file": "./run.tcl"
}
```

## 3. end_time 估算方法
- 查看目标 testbench（如 `circuit/s382_tb.v`）的 `initial` 时序：计算所有 `@(negedge/posedge CK)`、`#delay`、`repeat/for` 等耗时之和。
- s382 示例：
  - 时钟周期 10（`#5` 翻转，正负边各 5）。
  - 有 19 次 `@(negedge CK)` 激励序列后再 `#3 $finish;`。
  - 估算时间：19 个周期 × 10 ≈ 190，再加末尾 `#3`，可设 `end_time` 为 175~200 的安全窗口；示例使用 175（17.5 周期，满足全部激励并有余量）。
- 若不想细算，可将 `end_time` 设置为“(激励周期数+2)×clk_period”，稍大只会多耗少量仿真时间。

## 4. 运行仿真
在工程根目录执行（使用当前配置）:
```
D:\conda\python.exe py\simulator.py
```
流程：清理 -> 编译 PLI -> 黄金仿真 -> 生成黄金日志 -> 故障注入仿真 -> 计算 FDR 报告。

## 5. 切换检查清单
- verilog_file.f 仅包含目标电路及其 testbench、标准单元库。
- config.json 中的 `circuit_name`/`clk_name`/`clk_period`/`end_time` 已对应目标 testbench。
- 如更换电路同时更换 testbench 名称，确保 `verilog_file.f` 引用了正确的 tb，并与 `circuit_name` 匹配。

## 6. 常见问题
- 仿真未结束：`end_time` 太小或 testbench 无 `$finish`，适当放大 `end_time`。
- 编译缺文件：检查 `verilog_file.f` 路径及大小写。
- 时钟不匹配：确认 testbench 中的时钟信号名与 `config.json` 的 `clk_name` 一致。
