# Python_FDR 项目综合说明书

## 1. 项目简介
本项目是一套基于仿真的软错误率（Soft Error Rate, SER）分析工具，集成了硬件描述（Verilog）、C语言扩展（PLI/VPI）和 Python 自动化控制。它通过控制 EDA 仿真工具（如 VCS 或 Questasim/ModelSim），对数字电路进行故障注入（Fault Injection），并统计故障检测率（FDR, Fault Detection Rate）。

主要应用场景包括：
*   数字电路功能验证
*   故障仿真与抗辐照能力分析
*   EDA 工具流程开发与教学

## 3. 环境配置与依赖

### 3.1 操作系统支持
*   **Windows**: 推荐环境。支持 Questasim/ModelSim。需配置 GCC 环境（用于编译 C 语言 PLI 库）。

### 3.2 软件依赖
*   **Python**: 3.x
*   **仿真器**:
    *   Siemens Questasim / ModelSim (Windows)。推荐使用 **Questa/ModelSim 2020.4 SE** 或更高版本，以获得更好的 PLI/VPI 兼容性。
*   **编译器**:
    *   GCC (Windows 下需安装 MinGW 或类似环境，本项目 `gcc_toolchain` 已包含推荐版本)。

## 4. 快速开始

### 4.1 配置文件 (config.json)
在运行前，请确保根目录下有 `config.json` 文件，并根据实际环境修改：

```json
{
    "clk_name": "CK",              // 时钟信号名称
    "clk_period": 10,              // 时钟周期 (ns)
    "end_time": 132,               // 仿真结束时间 (ns)
    "circuit_info_file": "./output/circuit_info.json",
    "golden_file": "./output/golden.json",
    "fault_file": "./output/fault.json",
    "path": "./",                  // 项目根目录 (建议使用相对路径 "./")
    "tcl_file": "./run.tcl"        // 生成的 TCL 脚本路径
}
```

### 4.2 运行仿真

#### Windows (Questasim)
在 Windows 下，请直接运行 Python 控制脚本，它会自动调用批处理文件：

1.  **检查配置**: 打开 `compile_pli.bat` 和 `run_modelsim.bat`，确保 `MODELSIM_HOME` 指向您的 Questasim 安装路径（例如 `D:\questasim`）。
2.  **运行脚本**:
    ```powershell
    python py/simulator.py
    ```
    *该脚本会自动执行以下步骤：*
    *   调用 `compile_pli.bat` 编译 C 语言 PLI 库。
    *   生成 Golden Run 的 TCL 脚本。
    *   调用 `run_modelsim.bat` 运行仿真。
    *   生成故障注入 TCL 脚本并再次运行仿真。
    *   分析结果并生成报告。

## 5. 核心原理与算法

### 5.1 故障注入流程
程序通过生成 TCL 脚本控制仿真器：
1.  **Golden Run**: 运行无故障仿真，记录标准输出。
2.  **Fault Run**: 遍历电路中的寄存器，在不同时刻注入故障。
    *   重启仿真 (`restart`)
    *   调用 `$runfault` (PLI函数) 注入故障
    *   运行指定时间后，强制释放或修改信号 (`force ... -deposit`)
    *   运行至结束时间

### 5.2 FDR (故障检测率) 计算
`CircuitInfo` 类负责对比 Golden 和 Fault 结果：
*   **Error (Detected)**: 输出端口的值与 Golden 不一致。故障被系统检测到。
*   **Correct (Masked)**: 输出端口一致，且内部寄存器状态也一致。故障被逻辑屏蔽。
*   **Hide (Latent)**: 输出端口一致，但内部寄存器状态不一致。存在潜在故障，可能在后续周期显现。

计算公式：$FDR = \frac{Error}{Total\_Injections}$

## 6. 输出结果
运行结束后，程序会在控制台打印统计信息，并在 `output/` 目录下生成详细报告（支持 Excel/CSV 格式）。

```text
{
 'DFF_0': {'FDR': 0.15, 'correct': 80, 'error': 15, 'hide': 5},
 ...
}
```

## 7. 跨设备配置速查

| 项目 | 默认值 (示例) | 说明 |
| --- | --- | --- |
| `MODELSIM_HOME` | `D:\questasim` | `compile_pli.bat`、`run_modelsim.bat` 共用，需指向 Questasim/ModelSim 安装根目录。 |
| `VSIM_BIN` | `%MODELSIM_HOME%\win64` | `run_modelsim.bat` 自动加入 `PATH`，以调用 `vsim`/`vlog`。 |
| GCC 工具链 | `gcc_toolchain\gcc-4.5.0-mingw64vc12\bin` | `compile_pli.bat` 通过 `%~dp0` 相对路径引用，若迁移设备请更新该目录或保持相同结构。 |
| `config.json` | `项目根目录/config.json` | `py/simulator.py` 会使用脚本目录向上一层定位；若放置在其他位置，需要传入绝对路径。 |
| `run.tcl` | `./run.tcl` | Python 控制脚本会覆盖生成，保持与 `config.json` 中路径一致即可。 |
| 输出目录 | `./output` | 确保存在且具有写权限，Golden/Fault JSON 以及 Excel 报告都会写入。 |

**跨设备迁移建议**
- 保持 `gcc_toolchain/`、`compile_pli.bat`、`run_modelsim.bat` 位于项目根目录，脚本即可使用相对路径。
- 复制 Questasim 安装路径后，修改两个批处理脚本中 `MODELSIM_HOME` 即可。
- 若使用不同 Python 解释器，确保 `py/simulator.py` 中 `ensure_path` 能定位到根目录，必要时在 `config.json` 里把 `path` 改成绝对路径。
